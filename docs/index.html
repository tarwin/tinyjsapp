in ~6 MB.

A JavaScript backend with full system access, a native webview window (WebKit on macOS, WebView2 on Windows), and nothing else. No Electron. No bundled Chromium. No HTTP server. No ports.

Shipped app size

tinyjs
~6 MB

Your backend, a ~600 KB window process, and your plain HTML/CSS/JS. The window is the WebKit already on every Mac — or the WebView2 runtime already on every Windows 11. The Windows zip lands at ~6 MB too.

What you get

zero ports

Page ⇄ backend RPC over a Unix socket in a private temp dir. Nothing listens, nothing collides, nothing to scan.

full system access

Backend runs on txiki.js — files, sockets, processes, FFI, sqlite, fetch, WebSocket.

hot reload

Frontend edits swap into the live window; backend edits restart the process. No build step in dev.

native chrome

Real menu bar (About + Quit included), file panels, alert / confirm / prompt — answered by AppKit, not divs.

signed .app bundles

tinyjs build emits a codesigned, notarization-ready bundle with your icon.

agent-ready

Every project ships a skill file, so coding agents already know the whole API.

See it running

shelf — an app store for the examples

A 4.6 MB native app that shelves every example (pine by day, walnut by night) and installs, updates, and removes them in one click. The catalog updates itself from the examples repo — and now carries per-platform downloads, so it installs the Windows builds too.

⬇ Download shelf.zip · win 10/11 · in the examples repo ⬇ Download shelf.dmg · 4.6 MB · signed & notarized

All 25 examples were swept app-by-app on Windows: transparency, multi-window, drag-and-drop with real paths, tray apps (emoji: icons standing in for macOS sf: symbols), WebGPU, and media / WebAudio all work. The calls that are genuinely mac-only — Quick Look, AppleScript, Touch ID — answer 'unsupported', so a cross-platform app degrades honestly instead of breaking.

Quick start

terminal

tinyjs new myapp
cd myapp
tinyjs dev    # window opens, hot reload
tinyjs build  # dist/myapp.app, signed (win: dist/myapp.exe)

src/main.js — the whole backend

export const api = {
  hello: async ({ name }) =>
    `hi ${name}`,
};

in the page

await tiny.api.call('hello', { name });
tiny.menu.set([…]);
await tiny.win.openFile();
